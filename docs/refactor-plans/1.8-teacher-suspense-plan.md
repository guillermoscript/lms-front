# 1.8 Teacher Sub-Route Suspense Boundaries

## Goal
Add route-level `loading.tsx` files for teacher sub-routes so each page shows a layout-matched skeleton instead of falling back to the generic teacher dashboard skeleton.

## Analysis

| Route | Component Type | Layout Shape | Needs loading.tsx? |
|-------|---------------|-------------|-------------------|
| `courses/` | Server (async) | Header + search bar + card grid (4 cols) | Yes |
| `revenue/` | Server (async) | Header + 4 stat cards + split card + tabs | Yes |
| `community/` | Server (async) | Centered feed (max-w-3xl) with header bar | Yes |
| `templates/` | Client (`'use client'`) | Header + card with table (5 cols) | Yes (shell while JS loads) |
| `api-tokens/` | Server (async) | Simple wrapper, delegates to client component | Yes (light skeleton) |

All five sub-routes have layouts distinct from the generic teacher loading skeleton (which shows 3 stat cards + a table). Creating matched skeletons will eliminate the layout shift when navigating between these routes.

## Files to Create
1. `app/[locale]/dashboard/teacher/courses/loading.tsx`
2. `app/[locale]/dashboard/teacher/revenue/loading.tsx`
3. `app/[locale]/dashboard/teacher/community/loading.tsx`
4. `app/[locale]/dashboard/teacher/templates/loading.tsx`
5. `app/[locale]/dashboard/teacher/api-tokens/loading.tsx`

## Approach
- Use `Skeleton` from `@/components/ui/skeleton`
- Match outer container classes from each page (padding, max-width, grid columns)
- Include `role="status"` and `aria-busy="true"` for accessibility
- Keep skeletons minimal: approximate shapes, not pixel-perfect replicas
