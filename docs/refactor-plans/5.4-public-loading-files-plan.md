# 5.4 Public pages — loading.tsx files

## Problem
The `(public)` route group had zero `loading.tsx` files. All 11 pages are async server components with database queries, meaning users saw no visual feedback during navigation or initial load.

## Changes Made
Added `loading.tsx` with Skeleton-based placeholders to 5 routes:

1. `app/[locale]/(public)/courses/loading.tsx` — course catalog grid skeleton (dark theme matching page)
2. `app/[locale]/(public)/courses/[id]/loading.tsx` — course detail page skeleton
3. `app/[locale]/(public)/checkout/loading.tsx` — checkout form skeleton
4. `app/[locale]/(public)/products/loading.tsx` — product grid skeleton
5. `app/[locale]/(public)/pricing/loading.tsx` — pricing cards skeleton

## Routes not covered (intentionally)
- Landing page (`page.tsx`) — mostly static content, very fast
- `about/`, `creators/`, `platform-pricing/` — lightweight pages with minimal data fetching
- `p/[slug]/` — custom Puck pages have their own rendering pipeline

## Verification
`npx tsc --noEmit` passes with no errors.
