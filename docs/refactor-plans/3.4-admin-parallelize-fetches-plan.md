# 3.4 Admin Pages — Parallelize Fetches

## Status: DONE (partial — most were already parallelized)

## Assessment
- **admin/page.tsx** (dashboard): Already uses two `Promise.all` blocks for 9+ parallel queries. Well optimized.
- **courses/page.tsx**: Already uses `Promise.all` for authors, enrollments, lessons.
- **payment-requests/page.tsx**: Already parallelizes `getUserRole()`, `getCurrentTenantId()`, `isSuperAdmin()`.
- **subscriptions/page.tsx**: Single query with joins — no parallelization needed.

## Fixes Applied
1. **users/page.tsx**: Moved `enrollments` query and `getSchoolJoinUrl()` into a `Promise.all` block alongside the `profiles` query (all independent of each other, only depend on `memberUserIds`/`tenantId`).
2. **transactions/page.tsx**: Minor cleanup — deduplicated user IDs before profile lookup to avoid redundant fetches.
3. **products/[productId]/edit/page.tsx**: Parallelized product fetch with courses fetch using `Promise.all`.
4. **plans/[planId]/edit/page.tsx**: Parallelized plan fetch with courses fetch using `Promise.all`.
