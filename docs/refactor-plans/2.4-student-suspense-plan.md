# 2.4 Student Sub-Route Loading Skeletons

## Goal
Add route-specific `loading.tsx` files to all student dashboard sub-routes so Next.js can show skeleton UI during server-component data fetching via Suspense boundaries.

## Routes Addressed

| Route | Status | Skeleton Summary |
|-------|--------|-----------------|
| `student/browse/` | Created | Header + alert + search bar + 3x2 course card grid |
| `student/courses/` | Created | Header + filter pills + 4 horizontal course cards |
| `student/community/` | Created | Header bar + compose box + 3 post cards with avatar/reactions |
| `student/payments/` | Created | Header + alert + table (desktop) / cards (mobile) |
| `student/store/` | Created | Header + 3x2 store item grid |
| `student/profile/` | Created | Sidebar (profile card, gamification, subscription) + main (settings form, courses, billing) |
| `student/progress/` | Already existed (subtask 0.1) | Stats grid + course progress cards |
| `student/certificates/` | Already existed (subtask 0.1) | Header + stats row + certificate cards |

## Approach
- Each skeleton mirrors the actual page layout (container widths, grid columns, spacing)
- Uses `Skeleton` from `@/components/ui/skeleton` (animated pulse)
- Responsive: payments shows table skeleton on desktop, card skeletons on mobile
- Profile uses the two-column sidebar+main layout matching the actual page
