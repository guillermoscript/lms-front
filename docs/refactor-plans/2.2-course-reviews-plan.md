# 2.2 — Replace useEffect Data Fetching in course-reviews.tsx

## Problem

`CourseReviews` uses a `useEffect` to fetch reviews on mount via the Supabase client, causing a client-side waterfall:
1. Component mounts with empty `reviews` array + loading state
2. Fires 2 sequential client-side queries (reviews, then profiles for user info)
3. The parent server component can fetch this data server-side but currently doesn't

Additionally, the parent page's `userReview` query uses `.eq('course_id', ...)` and `.eq('tenant_id', ...)` — columns that don't exist on the `reviews` table (which uses `entity_type`/`entity_id` and has no `tenant_id`). This means `userHasReviewed` is always `false` from the server; the component re-derives it client-side.

## Approach

**Pass initial data from the server component; keep client-side refresh for mutations.**

- Reviews are interactive (add review) — must stay `'use client'`
- The server component can fetch reviews + profiles in a single pass
- After submitting a review, the existing `loadReviews()` handles refresh
- Fix the parent's broken `userReview` query while we're at it

## Changes

### 1. `course-reviews.tsx`
- Add `initialReviews: Review[]` to props interface
- Export the `Review` interface so the parent can use it
- Initialize `useState<Review[]>(initialReviews)` instead of `[]`
- Remove `loading` state — initial data is already available
- Remove the `useEffect` that calls `loadReviews()` on mount
- Keep `loadReviews()` as a private refresh function for post-mutation use

### 2. `page.tsx` (parent server component)
- Replace the broken `userReview` query with correct `entity_type`/`entity_id` filter
- Add a new query to fetch all reviews + profiles for the course
- Transform into `Review[]` shape and pass as `initialReviews`

## What stays the same
- The review submission form and mutation logic
- `loadReviews()` for post-mutation refresh
- The component remains `'use client'`
- All UI rendering logic
