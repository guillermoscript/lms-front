# 3.3 Admin Sub-Route loading.tsx Files

## Status: DONE

## Problem
Only 3 admin routes had `loading.tsx` files (admin root, products, enrollments). The remaining ~25 admin routes showed no loading state during SSR data fetching, causing a blank screen flash.

## Fix Applied
Created `loading.tsx` skeleton files for 14 high-traffic admin sub-routes:
- `courses/loading.tsx` — table + stats layout
- `users/loading.tsx` — user list with stats
- `transactions/loading.tsx` — table with financial stats
- `subscriptions/loading.tsx` — card list with filters
- `payment-requests/loading.tsx` — tabbed card list
- `plans/loading.tsx` — card grid (covers plans/new and plans/[id]/edit)
- `settings/loading.tsx` — form sections
- `notifications/loading.tsx` — list layout
- `categories/loading.tsx` — simple list
- `revenue/loading.tsx` — chart + stats
- `landing-page/loading.tsx` — page list
- `community/loading.tsx` — card list
- `billing/loading.tsx` — billing info sections
- `monetization/loading.tsx` — hub card grid

Each skeleton matches the actual page structure for minimal layout shift.

## Not Covered (parent loading.tsx applies)
- Deeply nested routes (e.g., `payment-requests/[requestId]`, `users/[userId]`)
- Edit sub-pages (covered by parent route's loading.tsx)
- Preview pages (ephemeral, low traffic)
