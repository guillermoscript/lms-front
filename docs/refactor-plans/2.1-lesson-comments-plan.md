# 2.1 — Replace useEffect Data Fetching in lesson-comments.tsx

## Problem

`LessonComments` uses a bare `useEffect` to fetch comments on mount, causing a client-side waterfall:
1. Component mounts with empty state + loading skeleton
2. Fires 3 sequential/parallel Supabase queries (comments, profiles, reactions)
3. The parent server component (`page.tsx`) **already fetches this exact data** via a joined query (line 54-57) but discards it

## Approach

**Pass initial data from the server component; keep client-side refresh for mutations.**

This is the right pattern because:
- Comments are interactive (add, delete, react) — must stay `'use client'`
- The server component already has the data — no need to re-fetch on mount
- After mutations (post, delete, react), the existing `loadComments()` function handles refresh
- No new dependencies needed (no SWR install)

## Changes

### 1. `lesson-comments.tsx`
- Add `initialComments: Comment[]` to props interface
- Initialize `useState<Comment[]>(initialComments)` instead of `[]`
- Remove `loading` state (initial data is already available)
- Remove the `useEffect` that calls `loadComments()` on mount
- Keep `loadComments()` as a private refresh function for post-mutation use
- Add a `refreshing` state for subtle refresh indicator (optional)

### 2. `page.tsx` (parent server component)
- Transform the already-fetched `lesson_comments` data into the `Comment[]` shape
- Pass it as `initialComments` prop to `<LessonComments>`

## What stays the same
- All mutation functions (post, delete, react)
- `loadComments()` for post-mutation refresh
- Sub-components (`CommentItem`, `CommentReplyForm`)
- The component remains `'use client'`
