# 0.2 Dynamic Imports for Heavy Components

**Rule:** `bundle-dynamic-imports` — Use `next/dynamic` to lazy-load large components not needed on initial render.
**Impact:** CRITICAL — directly affects TTI and LCP.

---

## Summary

9 components over 500 lines were identified as candidates. After analysis, **11 page/layout files** need changes across **8 distinct dynamic import targets**. Components in layouts or used on initial render unconditionally (e.g., `sidebar.tsx`) are excluded.

Additionally, **3 heavy-dependency components** under 500 lines are strong candidates due to their library weight (Puck, Sandpack, AI SDK).

## Changes

### Change 1: Lesson Editor Pages (2 files)

**Files:**
- `app/[locale]/dashboard/teacher/courses/[courseId]/lessons/new/page.tsx`
- `app/[locale]/dashboard/teacher/courses/[courseId]/lessons/[lessonId]/page.tsx`

**Why:** `LessonEditor` is 866 lines with block editor. These are server components that can use `next/dynamic` for the client child.

**Before** (both files):
```tsx
import { LessonEditor } from '@/components/teacher/lesson-editor'
```

**After** (both files):
```tsx
import dynamic from 'next/dynamic'
import { Skeleton } from '@/components/ui/skeleton'

const LessonEditor = dynamic(
  () => import('@/components/teacher/lesson-editor').then(m => m.LessonEditor),
  {
    ssr: false,
    loading: () => (
      <div className="mx-auto max-w-4xl p-6 space-y-6">
        <div className="flex items-center justify-between">
          <Skeleton className="h-8 w-48" />
          <Skeleton className="h-9 w-24" />
        </div>
        <Skeleton className="h-10 w-full" />
        <Skeleton className="h-32 w-full" />
        <Skeleton className="h-64 w-full" />
      </div>
    ),
  }
)
```

**Note:** `LessonEditorTour` is tiny and can remain static.

### Change 5: Student Exercise Page — AudioExercise, CodeChallengeWrapper, ArtifactExercise (1 file)

**File:** `app/[locale]/dashboard/student/courses/[courseId]/exercises/[exerciseId]/page.tsx`

All three are conditionally rendered based on `exercise.exercise_type`. Dynamic import avoids loading all three when only one is needed.

**Before:**
```tsx
import AudioExercise from '@/components/exercises/audio-exercise'
import CodeChallengeWrapper from '@/components/exercises/code-challenge-wrapper'
import ArtifactExercise from '@/components/exercises/artifact-exercise'
```

**After:**
```tsx
import dynamic from 'next/dynamic'
import { Skeleton } from '@/components/ui/skeleton'

const AudioExercise = dynamic(
  () => import('@/components/exercises/audio-exercise'),
  {
    ssr: false,
    loading: () => (
      <div className="space-y-4 p-6">
        <Skeleton className="h-8 w-64" />
        <Skeleton className="h-48 w-full rounded-xl" />
        <Skeleton className="h-12 w-32 mx-auto" />
      </div>
    ),
  }
)

const CodeChallengeWrapper = dynamic(
  () => import('@/components/exercises/code-challenge-wrapper'),
  {
    ssr: false,
    loading: () => (
      <div className="space-y-2">
        <Skeleton className="h-8 w-full" />
        <Skeleton className="h-[400px] w-full" />
      </div>
    ),
  }
)

const ArtifactExercise = dynamic(
  () => import('@/components/exercises/artifact-exercise'),
  {
    ssr: false,
    loading: () => (
      <div className="space-y-4 p-6">
        <Skeleton className="h-8 w-64" />
        <Skeleton className="h-64 w-full rounded-xl" />
      </div>
    ),
  }
)
```

**Note:** `CodeChallengeWrapper` imports `@codesandbox/sandpack-react` (~300KB). This is the highest-impact single change — it prevents Sandpack from loading for audio/essay/artifact exercises.

### Change 6: PuckEditor inside LandingPagesClient (1 file — client component)

**File:** `components/admin/landing-page/landing-pages-client.tsx`

This is a **client component** that conditionally renders `PuckEditor` only when `editingPage` is set. The Puck library (`@measured/puck`) is ~200KB+ and should not load until the user clicks "Edit".

**Before:**
```tsx
import { PuckEditor } from './puck-editor'
```

**After:**
```tsx
import dynamic from 'next/dynamic'

const PuckEditor = dynamic(
  () => import('./puck-editor').then(m => m.PuckEditor),
  {
    ssr: false,
    loading: () => (
      <div className="flex items-center justify-center min-h-screen">
        <div className="animate-spin h-8 w-8 border-2 border-primary border-t-transparent rounded-full" />
      </div>
    ),
  }
)
```

### Change 7: AristotlePanel in Student Course Layout (1 file)

**File:** `app/[locale]/dashboard/student/courses/[courseId]/layout.tsx`

The `AristotlePanel` (253 lines) transitively imports `prompt-input.tsx` (1,263 lines) and AI SDK. It renders in every student course page but is hidden until the user clicks the trigger button. This is a strong candidate for lazy loading since it's behind user interaction.

**Before:**
```tsx
import { AristotlePanel } from '@/components/aristotle/aristotle-panel'
```

**After:**
```tsx
import dynamic from 'next/dynamic'

const AristotlePanel = dynamic(
  () => import('@/components/aristotle/aristotle-panel').then(m => m.AristotlePanel),
  { ssr: false }
)
```

**No loading skeleton needed** — the panel is invisible until toggled, and `AristotleTrigger` (the button) remains statically imported.

## Excluded Candidates

| Component | Reason |
|-----------|--------|
| `components/ui/sidebar.tsx` | Layout shell component, used in every dashboard page. Dynamic import would cause layout shift. |
| `components/ai-elements/prompt-input.tsx` | Never imported directly by page files — only transitively via components that are already being dynamically imported. |
| `components/ai-elements/code-block.tsx` | Never imported directly by page files — transitive only. |
| `components/component-example.tsx` | Not imported by any file in the codebase. Dead code. |
| `components/public/landing-page/puck-page-renderer.tsx` | Only 21 lines. The Puck `Render` component it wraps is lighter than `Puck` editor. Used on public landing pages where it IS the main content (not behind interaction). |
