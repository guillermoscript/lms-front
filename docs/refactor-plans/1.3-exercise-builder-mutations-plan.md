## Mutations Found

The component has a single `handleSave` function (lines 107-174) that performs **two mutations** depending on context:

### Mutation 1: Create Exercise (INSERT)

- **Table:** `exercises`
- **When:** `initialData` is falsy (new exercise mode)
- **Data written:** `{ course_id, lesson_id, title, description, instructions, exercise_type, difficulty_level, time_limit, system_prompt, status, created_by, exercise_config? }`
- **Auth needed:** Teacher/admin who owns the course
- **After success (draft):** Sets `saveSuccess = true`, stays on page
- **After success (publish):** Redirects to `/dashboard/teacher/courses/${courseId}/exercises`, calls `router.refresh()`

**Current code (lines 155-159):**
```typescript
const { error: insertError } = await supabase
  .from('exercises')
  .insert([exerciseData])
if (insertError) throw insertError
```

### Mutation 2: Update Exercise (UPDATE)

- **Table:** `exercises`
- **When:** `initialData` is truthy (edit mode)
- **Data written:** Same fields as create
- **Auth needed:** Teacher/admin who owns the course
- **After success (draft):** Sets `saveSuccess = true`, stays on page
- **After success (publish):** Redirects to `/dashboard/teacher/courses/${courseId}/exercises`, calls `router.refresh()`

**Current code (lines 150-154):**
```typescript
const { error: updateError } = await supabase
  .from('exercises')
  .update(exerciseData)
  .eq('id', initialData.id)
if (updateError) throw updateError
```

### Audio/Video Config (conditional, embedded in both mutations)

When `exercise_type` is `audio_evaluation` or `video_evaluation`, an `exercise_config` JSONB column is populated with:
```typescript
{
  stt_provider: 'assemblyai',
  ai_coach: 'openai',
  topic_prompt, min_duration_seconds, max_duration_seconds,
  passing_score, max_daily_attempts,
  rubric: { filler_words, pace, structure, confidence }
}
```

This is not a separate mutation -- it is a field on the same `exercises` row. Handled within the same insert/update.

## Security Issues in Current Code

| Issue | Details |
|-------|---------|
| **No server-side auth** | Component calls `supabase.auth.getUser()` on the client but does not verify role or ownership server-side. Relies entirely on RLS. |
| **No course ownership check** | Any authenticated user with RLS INSERT on `exercises` could create exercises on any course in their tenant. |
| **No tenant_id filter on update** | `.update(exerciseData).eq('id', initialData.id)` has no `.eq('tenant_id', ...)`. Relies on RLS only. |
| **`created_by` set on client** | `created_by: user.id` is set client-side. Should be set server-side from the authenticated user. |

## Server Action Design

### File: `app/actions/teacher/exercises.ts`

Since both mutations happen in `handleSave`, we consolidate into **two server actions** (create and update). The audio/video config is assembled server-side from the same form data.

#### Action 1: `createExercise(courseId: number, data: ExerciseFormData)`

Returns `ActionResult<{ exerciseId: number }>`.

```typescript
export async function createExercise(
  courseId: number,
  data: ExerciseFormData
) {
  return actionHandler(async () => {
    const ctx = await requireTeacherOrAdmin()
    await verifyCourseOwnership(ctx, courseId)

    if (!data.title?.trim()) throw new Error('Title is required')

    const isAudioType = data.exercise_type === 'audio_evaluation' || data.exercise_type === 'video_evaluation'

    const exerciseData: Record<string, unknown> = {
      course_id: courseId,
      tenant_id: ctx.tenantId,
      lesson_id: data.lesson_id || null,
      title: data.title.trim(),
      description: data.description || null,
      instructions: data.instructions || null,
      exercise_type: data.exercise_type,
      difficulty_level: data.difficulty_level,
      time_limit: data.time_limit,
      system_prompt: data.system_prompt || null,
      status: data.publish ? 'published' : data.status,
      created_by: ctx.userId,
    }

    if (isAudioType) {
      exerciseData.exercise_config = buildAudioConfig(data)
    }

    const { data: newExercise, error } = await ctx.supabase
      .from('exercises')
      .insert(exerciseData)
      .select('id')
      .single()

    if (error) throw error

    revalidatePath(`/dashboard/teacher/courses/${courseId}/exercises`)

    return { exerciseId: newExercise.id }
  })
}
```

## Security Improvements

| Concern | Before | After |
|---------|--------|-------|
| **Auth check** | Client-side `getUser()` only -- no role verification | `requireTeacherOrAdmin()` verifies role server-side |
| **Course ownership** | None -- any teacher in the tenant could create exercises on any course | `verifyCourseOwnership(ctx, courseId)` checks author_id (teacher) or tenant membership (admin) |
| **Tenant isolation** | RLS only (no explicit `tenant_id` filter on update) | Explicit `.eq('tenant_id', ctx.tenantId)` on update query; explicit `tenant_id` set on insert |
| **`created_by` spoofing** | Set from client-side `user.id` -- could be manipulated | Set from `ctx.userId` (server-verified) |
| **Input validation** | None | Title required check server-side |
