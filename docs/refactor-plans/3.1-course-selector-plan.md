# 3.1 CourseSelector — useEffect fetch to server-provided props

## Status: DONE

## Problem
`components/admin/course-selector.tsx` used a `useEffect` to fetch courses from Supabase client-side. This caused:
- Waterfall: page renders, then client JS loads, then fetch fires
- Missing `tenant_id` filter (relied on RLS only)
- Unnecessary client bundle weight (`@supabase/supabase-js` client)

## Fix Applied
1. Removed `useEffect`, `useState` for courses/loading, and `createClient` import from `CourseSelector`
2. Added `courses: CourseOption[]` prop — data now comes from the server
3. Updated `ProductForm` and `PlanForm` to accept and pass `courses` prop
4. Updated all 4 parent server pages to fetch published courses server-side:
   - `products/new/page.tsx` — added course query with tenant filter
   - `products/[productId]/edit/page.tsx` — parallelized with `Promise.all`
   - `plans/new/page.tsx` — added course query with tenant filter
   - `plans/[planId]/edit/page.tsx` — parallelized with `Promise.all`

## Files Changed
- `components/admin/course-selector.tsx`
- `components/admin/product-form.tsx`
- `components/admin/plan-form.tsx`

- `app/[locale]/dashboard/admin/plans/new/page.tsx`
