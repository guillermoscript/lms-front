# 0.3 Split useGamification Hook

## Problem

`lib/hooks/use-gamification.ts` is a 260-line "god hook" with **13 return values** and **4 loading states**. Every consumer instantiates the full hook but only uses 2-4 of its return values. This means:

- Every component that needs just `summary` also creates state for `leaderboard`, `allAchievements`, and `storeItems`
- Each component gets its own independent copy of the hook state (no sharing)
- Components that only show XP/level still allocate memory for store items and achievements
- The `useEffect` in the hook calls `fetchSummary()` on mount for every consumer, causing redundant network requests

## Current Hook Analysis

### All 13 Return Values (categorized)

| # | Return Value | Type | Data Source | Category |
|---|---|---|---|---|
| 1 | `summary` | `GamificationSummary \| null` | Edge function `get-gamification-summary` | **Summary/XP** |
| 2 | `loading` | `boolean` | Summary fetch state | **Summary/XP** |
| 3 | `error` | `Error \| null` | Summary fetch error | **Summary/XP** |
| 4 | `refresh` | `() => Promise<void>` | Re-fetches summary | **Summary/XP** |
| 5 | `leaderboard` | `LeaderboardEntry[]` | Edge function `get-leaderboard` | **Leaderboard** |
| 6 | `leaderboardLoading` | `boolean` | Leaderboard fetch state | **Leaderboard** |
| 7 | `refreshLeaderboard` | `(timeframe, limit) => Promise<void>` | Fetches leaderboard | **Leaderboard** |
| 8 | `allAchievements` | `Achievement[]` | `gamification_achievements` + `gamification_user_achievements` tables | **Achievements** |
| 9 | `achievementsLoading` | `boolean` | Achievements fetch state | **Achievements** |
| 10 | `refreshAchievements` | `() => Promise<void>` | Fetches achievements | **Achievements** |
| 11 | `storeItems` | `StoreItem[]` | `gamification_store_items` table | **Store** |
| 12 | `storeLoading` | `boolean` | Store fetch state | **Store** |
| 13 | `refreshStore` | `() => Promise<void>` | Fetches store items | **Store** |
| -- | `purchaseItem` | `(itemId) => Promise<Result>` | Edge function `spend-points` | **Store** |

Note: `purchaseItem` is the 14th value (the task description said 13, but there are actually 14).

### 4 Independent Data Domains

1. **Summary/XP** -- auto-fetched on mount via `useEffect`. Contains XP, level, streak, coins, features, recent achievements. Used by most consumers.
2. **Leaderboard** -- lazy-loaded (only fetched when `refreshLeaderboard()` is called). Completely independent data source.
3. **Achievements** -- lazy-loaded. Joins `gamification_achievements` with `gamification_user_achievements`.
4. **Store** -- lazy-loaded. Queries `gamification_store_items` and provides `purchaseItem` mutation (which also refreshes summary for coin balance update).

## Proposed Split: 4 Focused Hooks

### 1. `useGamificationSummary()` -- Core summary data

**Responsibility:** Fetches the gamification summary (XP, level, streak, coins, features). Auto-fetches on mount.

**Return interface:**
```typescript
interface UseGamificationSummaryReturn {
  summary: GamificationSummary | null;
  loading: boolean;
  error: Error | null;
  refresh: () => Promise<void>;
}
```

**Consumers:** `GamificationHeaderCard`, `XPProgressCircle`, `ProfileGamificationStats`, `StreakCalendar`, and indirectly by `MiniLeaderboard`, `AchievementGrid`, `StoreSection`, `PointStoreItem` (for feature gating / coin balance).

**File:** `lib/hooks/use-gamification-summary.ts`

### 2. `useLeaderboard()` -- Leaderboard data

**Responsibility:** Fetches leaderboard rankings. Lazy-loaded (call `fetch` to load).

**Return interface:**
```typescript
interface UseLeaderboardReturn {
  leaderboard: LeaderboardEntry[];
  loading: boolean;
  fetch: (timeframe?: "weekly" | "monthly" | "all_time", limit?: number) => Promise<void>;
}
```

**Consumers:** `MiniLeaderboard`

**File:** `lib/hooks/use-leaderboard.ts`

### 3. `useAchievements()` -- Achievement data

**Responsibility:** Fetches all achievements with earned status. Lazy-loaded.

**Return interface:**
```typescript
interface UseAchievementsReturn {
  achievements: Achievement[];
  loading: boolean;
  fetch: () => Promise<void>;
}
```

**Consumers:** `AchievementGrid`

**File:** `lib/hooks/use-achievements.ts`

### 4. `usePointStore()` -- Store items + purchase

**Responsibility:** Fetches available store items and provides purchase action. Lazy-loaded.

**Return interface:**
```typescript
interface UsePointStoreReturn {
  items: StoreItem[];
  loading: boolean;
  fetch: () => Promise<void>;
  purchase: (itemId: string) => Promise<{ success: boolean; error?: string; data?: any }>;
}
```

**Consumers:** `StoreSection`, `PointStoreItem`

**File:** `lib/hooks/use-point-store.ts`

### Design Decisions

1. **`summary` stays independent** -- not lifted into context. Each component that calls `useGamificationSummary()` gets its own copy. This is acceptable because: (a) the profile page has 4 consumers but they mount together and the browser caches the edge function response, (b) lifting to context would require a provider wrapper and adds complexity not justified by the current usage pattern. If performance becomes an issue later, summary can be lifted to a `GamificationProvider` context.

2. **`purchaseItem` refreshes summary** -- the `usePointStore` hook accepts an optional `onPurchase` callback so the consumer can trigger a summary refresh after purchase. Alternatively, `PointStoreItem` can call both `useGamificationSummary()` and `usePointStore()`.

3. **Feature gating stays in consumers** -- Components like `MiniLeaderboard` need `summary.features.leaderboard` to decide whether to render. They will import `useGamificationSummary` alongside their domain-specific hook.

4. **Types stay exported from a shared file** -- `GamificationSummary`, `LeaderboardEntry`, `Achievement`, `StoreItem`, `GamificationFeatures` move to `lib/hooks/use-gamification-types.ts` (or stay in individual hook files where they're used).

```typescript
"use client";

import { useEffect, useState } from "react";
import { createClient } from "@/lib/supabase/client";

export interface GamificationFeatures {
  xp: boolean;
  levels: boolean;
  streaks: boolean;
  leaderboard: boolean;
  achievements: boolean;
  store: boolean;
  custom_achievements: boolean;
  custom_store: boolean;
}

export interface GamificationSummary {
  total_xp: number;
  level: number;
  level_title: string;
  level_icon: string | null;
  xp_progress: {
    current: number;
    required: number;
    percentage: number;
  };
  streak: {
    current: number;
    longest: number;
    freezes_available: number;
    last_activity: string | null;
  };
  coins: number;
  achievements: {
    earned: number;
    total: number;
    recent: any[];
    newly_earned: string[];
  };
  recent_xp: any[];
  features: GamificationFeatures;
}

export function useGamificationSummary() {
  const [summary, setSummary] = useState<GamificationSummary | null>(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<Error | null>(null);
  const supabase = createClient();

  const fetchSummary = async () => {
    try {
      setLoading(true);
      const { data: { session } } = await supabase.auth.getSession();

      if (!session) {
        setLoading(false);
        return;
      }

      const response = await fetch(
        `${process.env.NEXT_PUBLIC_SUPABASE_URL}/functions/v1/get-gamification-summary`,
        {
          headers: {
            Authorization: `Bearer ${session.access_token}`,
          },
        }
      );

      if (!response.ok) {
        console.warn("Gamification summary unavailable (status %d), using defaults", response.status);
        setSummary(null);
        return;
      }

      const data = await response.json();
      setSummary(data);
    } catch (err: any) {
      console.warn("Gamification fetch failed, using defaults:", err.message);
      setSummary(null);
    } finally {
      setLoading(false);
    }
  };

  useEffect(() => {
    fetchSummary();
  }, []);

  return { summary, loading, error, refresh: fetchSummary };
}
```

"use client";

import { useState } from "react";
import { createClient } from "@/lib/supabase/client";

export interface LeaderboardEntry {
  rank: number;
  user_id: string;
  full_name: string | null;
  avatar_url: string | null;
  username: string | null;
  xp: number;
  level: number;
  current_streak: number;
}

export function useLeaderboard() {
  const [leaderboard, setLeaderboard] = useState<LeaderboardEntry[]>([]);
  const [loading, setLoading] = useState(false);
  const supabase = createClient();

  const fetch = async (timeframe: "weekly" | "monthly" | "all_time" = "weekly", limit = 5) => {
    try {
      setLoading(true);
      const { data: { session } } = await supabase.auth.getSession();

      if (!session) {
        setLoading(false);
        return;
      }

      const response = await globalThis.fetch(
        `${process.env.NEXT_PUBLIC_SUPABASE_URL}/functions/v1/get-leaderboard?timeframe=${timeframe}&limit=${limit}`,
        {
          headers: {
            Authorization: `Bearer ${session.access_token}`,
          },
        }
      );

      if (!response.ok) {
        const errorData = await response.json().catch(() => ({}));
        console.error("Leaderboard error:", errorData);
        throw new Error(errorData.error || "Failed to fetch leaderboard");
      }

      const data = await response.json();
      setLeaderboard(data);
    } catch (err: any) {
      console.error(err);
    } finally {
      setLoading(false);
    }
  };

  return { leaderboard, loading, fetch };
}

#### `lib/hooks/use-achievements.ts` (NEW)

```typescript
"use client";

import { useState } from "react";
import { createClient } from "@/lib/supabase/client";

export interface Achievement {
  id: string;
  slug: string;
  title: string;
  description: string;
  tier: "bronze" | "silver" | "gold" | "platinum";
  category: string;
  icon: string;
  xp_reward: number;
  earned_at?: string;
}

export function useAchievements() {
  const [achievements, setAchievements] = useState<Achievement[]>([]);
  const [loading, setLoading] = useState(false);
  const supabase = createClient();

  const fetch = async () => {
    try {
      setLoading(true);
      const { data: { user } } = await supabase.auth.getUser();
      if (!user) return;

      const [achievementsRes, userAchievementsRes] = await Promise.all([
        supabase.from("gamification_achievements").select("*").order("tier", { ascending: true }),
        supabase.from("gamification_user_achievements").select("*").eq("user_id", user.id)
      ]);

      if (achievementsRes.error) throw achievementsRes.error;

      const earnedMap = new Map(
        (userAchievementsRes.data || []).map(ua => [ua.achievement_id, ua.earned_at])
      );

      const combined: Achievement[] = (achievementsRes.data || []).map(a => ({
        ...a,
        earned_at: earnedMap.get(a.id)
      }));

      setAchievements(combined);
    } catch (err: any) {
      console.error(err);
    } finally {
      setLoading(false);
    }
  };

  return { achievements, loading, fetch };
}
```

#### `lib/hooks/use-point-store.ts` (NEW)

```typescript
"use client";

import { useState } from "react";
import { createClient } from "@/lib/supabase/client";

export interface StoreItem {
  id: string;
  slug: string;
  name: string;
  description: string;
  price_coins: number;
  category: string;
  icon: string;
  metadata: any;
  is_available: boolean;
}

export function usePointStore(options?: { onPurchase?: () => void }) {
  const [items, setItems] = useState<StoreItem[]>([]);
  const [loading, setLoading] = useState(false);
  const supabase = createClient();

  const fetchItems = async () => {
    try {
      setLoading(true);
      const { data, error } = await supabase
        .from("gamification_store_items")
        .select("*")
        .eq("is_available", true);

      if (error) throw error;
      setItems(data || []);
    } catch (err: any) {
      console.error(err);
    } finally {
      setLoading(false);
    }
  };

  const purchase = async (itemId: string) => {
    try {
      const { data: { session } } = await supabase.auth.getSession();
      if (!session) return { success: false, error: "Not authenticated" };

      const response = await globalThis.fetch(
        `${process.env.NEXT_PUBLIC_SUPABASE_URL}/functions/v1/spend-points`,
        {
          method: "POST",
          headers: {
            Authorization: `Bearer ${session.access_token}`,
            "Content-Type": "application/json",
          },
          body: JSON.stringify({ item_id: itemId }),
        }
      );

      const result = await response.json();
      if (response.ok) {
        options?.onPurchase?.();
        return { success: true, data: result };
      } else {
        return { success: false, error: result.error };
      }
    } catch (err: any) {
      return { success: false, error: err.message };
    }
  };

  return { items, loading, fetch: fetchItems, purchase };
}
```

## Migration Strategy

1. **Create the 4 new hook files** -- no consumers change yet.
2. **Update `use-gamification.ts` to be the facade** -- re-exports types and new hooks, plus deprecated `useGamification` alias.
3. **Migrate consumers one at a time**, starting with the simplest (summary-only: `GamificationHeaderCard`, `XPProgressCircle`, `ProfileGamificationStats`, `StreakCalendar`).
4. **Migrate complex consumers** (`MiniLeaderboard`, `AchievementGrid`, `StoreSection`, `PointStoreItem`).
5. **Verify** `npm run build` passes after each step.
6. **Remove the deprecated facade** in a follow-up if desired (low priority since it's just re-exports).

## Impact Assessment

| Metric | Before | After |
|---|---|---|
| Hook files | 1 (260 lines) | 4 focused hooks (~60-80 lines each) + 1 facade (~20 lines) |
| Return values per hook | 14 | 4, 3, 3, 4 |
| Loading states per hook | 4 | 1 each |
| State allocated by summary-only consumers | 8 `useState` calls (4 data + 4 loading) | 3 `useState` calls (1 data + 1 loading + 1 error) |
| Redundant data fetching | Yes (all hooks fetch summary independently) | Same (acceptable; see Design Decision #1) |
| Lines changed in consumers | 0 | ~2-6 per file (import + destructure) |

## Future Enhancement: GamificationProvider Context

If the profile page performance becomes an issue (4 components each calling `useGamificationSummary` = 4 edge function calls), wrap them in a shared provider:

```typescript
// lib/contexts/gamification-context.tsx
const GamificationContext = createContext<UseGamificationSummaryReturn | null>(null);

export function GamificationProvider({ children }: { children: React.ReactNode }) {
  const value = useGamificationSummary();
  return <GamificationContext value={value}>{children}</GamificationContext>;
}

export function useGamificationContext() {
  const ctx = use(GamificationContext);
  if (!ctx) throw new Error("useGamificationContext must be within GamificationProvider");
  return ctx;
}
```

This follows the `state-context-interface` pattern. Not needed now, but the hook split makes it easy to add later.
