# 3.5 Landing Page Components — Server/Client Split

## Status: SKIPPED (not actionable)

## Assessment
The Puck visual editor (`@measured/puck`) is inherently a client-side library:
- `PuckEditor` component wraps `<Puck>` which requires DOM access for drag-and-drop
- `PuckPageRenderer` wraps `<Render>` with `DropZone` which also needs client-side rendering
- Both are already marked `'use client'`

The parent page (`landing-page/page.tsx`) is a server component that fetches data and passes it to `LandingPagesClient`. `LandingPagesClient` already uses `next/dynamic` to lazy-load PuckEditor.

The current architecture is already well-structured:
- Server component handles data fetching (pages, templates, tenant info)
- Client component handles interactive UI (editing, template picking)
- Heavy editor is dynamically imported

No meaningful server/client split optimization is possible here.
