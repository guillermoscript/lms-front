# 2.3 Browse Card Variants Refactor

## Problem

`BrowseCourseCard` has 3 boolean props (`isEnrolled`, `hasActiveSubscription`, `isCoveredByPlan`) plus an associated `subscriptionId` that together control a 4-way conditional in the footer and a 2-way conditional for link destinations. This is the boolean prop proliferation anti-pattern -- the booleans encode 4 discrete states implicitly.

## Solution

Replace the 3 booleans + subscriptionId with a single discriminated union prop `enrollmentStatus`:

```typescript
type EnrollmentStatus =
  | { variant: 'enrolled' }
  | { variant: 'enrollable'; subscriptionId: number }
  | { variant: 'not-in-plan' }
  | { variant: 'no-subscription' }
```

This makes impossible states unrepresentable (e.g., `isEnrolled=true` + `hasActiveSubscription=false` + `isCoveredByPlan=true` was technically possible but meaningless).

## Files Changed

- `components/student/browse-course-card.tsx` -- replace boolean props with discriminated union
- `app/[locale]/dashboard/student/browse/page.tsx` -- update callsite to compute variant
