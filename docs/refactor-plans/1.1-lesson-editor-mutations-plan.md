## Mutations Found

The component has a single `handleSave` function (lines 116-182) that performs **three mutations** depending on context:

### Mutation 1: Create Lesson (INSERT)

- **Table:** `lessons`
- **When:** `initialData` is falsy (new lesson mode)
- **Data written:** `{ course_id, title, description, content, video_url, sequence, status, publish_at }`
- **Auth needed:** Teacher/admin who owns the course
- **After success (draft):** Sets `saveSuccess = true`, stays on page
- **After success (publish):** Redirects to `/dashboard/teacher/courses/${courseId}`

**Current code (lines 142-151):**
```typescript
const { data: newLesson, error: insertError } = await supabase
  .from('lessons')
  .insert([lessonData])
  .select('id')
  .single()
if (insertError) throw insertError
lessonId = newLesson.id
```

### Mutation 2: Update Lesson (UPDATE)

- **Table:** `lessons`
- **When:** `initialData` is truthy (edit mode)
- **Data written:** Same fields as create
- **Auth needed:** Teacher/admin who owns the course
- **After success (draft):** Sets `saveSuccess = true`, stays on page
- **After success (publish):** Redirects to `/dashboard/teacher/courses/${courseId}`

**Current code (lines 136-141):**
```typescript
const { error: updateError } = await supabase
  .from('lessons')
  .update(lessonData)
  .eq('id', initialData.id)
if (updateError) throw updateError
```

### Mutation 3: Upsert AI Task (UPSERT)

- **Table:** `lessons_ai_tasks`
- **When:** After lesson save, if `ai_task_description` or `ai_task_instructions` is non-empty
- **Data written:** `{ lesson_id, task_instructions, system_prompt }` (on conflict: `lesson_id`)
- **Auth needed:** Same as lesson save (teacher/admin who owns the course)
- **Note:** This is always performed as part of the same save operation, never independently

**Current code (lines 153-169):**
```typescript

    {

      system_prompt: formData.ai_task_instructions || '',
    },
    { onConflict: 'lesson_id' }
  )
if (taskError) throw taskError
```

## Server Action Design

### File: `app/actions/teacher/lessons.ts`

Since mutations 1-3 always happen together in `handleSave`, we consolidate into **two server actions** (create and update) rather than three. The AI task upsert is an internal step of both.

#### Types

```typescript
interface LessonFormData {
  title: string
  description: string
  content: string
  video_url: string
  sequence: number
  publish: boolean
  publish_at: string
  ai_task_description: string
```

#### Action 1: `createLesson(courseId: number, data: LessonFormData)`

Returns `ActionResult<{ lessonId: number }>`.

```typescript
export async function createLesson(
  courseId: number,
  data: LessonFormData
) {
  return actionHandler(async () => {
    const ctx = await requireTeacherOrAdmin()
    await verifyCourseOwnership(ctx, courseId)

    if (!data.title?.trim()) throw new Error('Title is required')

    const isScheduled = !data.publish && data.publish_at
    const { data: newLesson, error } = await ctx.supabase
      .from('lessons')
      .insert({
        course_id: courseId,
        title: data.title.trim(),
        description: data.description || null,
        content: data.content || null,
        video_url: data.video_url || null,
        sequence: data.sequence,
        status: data.publish ? 'published' : 'draft',
        publish_at: isScheduled ? data.publish_at : null,
      })
      .select('id')
      .single()

    if (error) throw error

    // Upsert AI task if provided
    if (data.ai_task_description || data.ai_task_instructions) {
      const { error: taskError } = await ctx.supabase
        .from('lessons_ai_tasks')
        .upsert(
          {
            lesson_id: newLesson.id,
            task_instructions: data.ai_task_description || '',
            system_prompt: data.ai_task_instructions || '',
          },
          { onConflict: 'lesson_id' }
        )
      if (taskError) throw taskError
    }

    revalidatePath(`/dashboard/teacher/courses/${courseId}`)

    return { lessonId: newLesson.id }
  })
}
```

## Security Improvements

| Concern | Before | After |
|---------|--------|-------|
| **Auth check** | None -- relies on RLS and page-level guards | Explicit `requireTeacherOrAdmin()` inside each action |
| **Course ownership** | None in component (page checks, but client mutation bypasses) | `verifyCourseOwnership(ctx, courseId)` in each action |
| **Tenant isolation** | RLS only (no explicit `tenant_id` filter on update) | Explicit `.eq('tenant_id', ctx.tenantId)` on update query |
| **Input validation** | None | Title required check; fields trimmed/nullified |

The current code's `.update(lessonData).eq('id', initialData.id)` has **no tenant_id filter** -- it relies entirely on RLS. The server action adds an explicit `.eq('tenant_id', ctx.tenantId)`.
